## Известные проблемы

- **`WithSortAndFilter` работает на уже отфильтрованном `cv.items`** — при активном `UnreadOnly` функция получает срез `cv.items`, из которого уже удалены каналы с нулевым счётчиком. Когда такой канал получает новое сообщение через WS и его счётчик становится > 0, он не может вернуться в список — он уже потерян. Кроме того, `[channels] unread_only = true` в конфиге приводит к тому, что список каналов навсегда остаётся пустым. Исправление: всегда перефильтровывать из полного канонического списка (`m.channelsRaw`). Источник: `channels_view.go:265`, `model.go:WithSortAndFilter`.
- **`loadUnreadsCmd` не принимает контекст** — в отличие от `resolveDMNames`, эта функция не имеет контекста и не может быть отменена при выходе. На больших серверах с сотнями каналов горутина и HTTP-соединения остаются открытыми после `/quit`. Исправление: передать `m.ctx` аналогично `resolveDMNames`. Источник: `model.go:676`.
- **Spinner — первый кадр при повторном переключении каналов** — при переключении на второй канал placeholder показывает кадр спиннера, замороженный после предыдущей загрузки, до прихода первого нового тика (~1/12 с). Источник: `model.go` `MsgChannelSelected`.
- **All Activity без store** — если `m.store == nil` (тесты или деградированный режим), возврат в «All Activity» оставляет ленту пустой, даже если сообщения приходили через WS. Источник: `model.go` `MsgChannelSelected`, ветка `ChannelID == ""`.
- **`/reset db` без подтверждения** — команда необратимо удаляет локальную БД без undo и без подсказки. Стоит добавить hint в статус-баре после выполнения.
- **Бейджи тредов не отображаются при первой загрузке канала через REST** — когда сообщения загружаются впервые (канал ещё не в кеше), значки «N ответов» на сообщениях не появляются. Причина: REST `/channels/{id}/posts` возвращает `reply_count` в каждом посте, но `MsgChannelHistory` не проставляет это поле в `feedItem` при первом рендере. После прихода нового поста или перезагрузки страницы бейджи появляются. Источник: `model.go MsgChannelHistory`, `messages_view.go`.

## Performance / Архитектура

- **High-water mark: загрузка дельты при холодном старте** — сейчас All Activity при старте показывает 100 последних сообщений из локального кеша, но не подгружает новые сообщения, которые пришли пока приложение было закрыто. Решение: хранить в БД `last_seen_at` (новая таблица `metadata`), при старте автоматически вызывать REST `/channels/{id}/posts?since=<last_seen>` для каждого канала и мёржить с кешем. Это устранит "дыру" во временной шкале All Activity. Источник: анализ `model.go:MsgHistoryLoaded`, `store/db.go:GetRecentMessages`, обсуждение 2026-05-19.

## Функциональность

- **История навигации (back/forward)** — хранить стек открытых каналов/тредов в текущей сессии; `Alt+←` / `Alt+→` переходят назад и вперёд по истории аналогично браузеру. Актуально когда пользователь переходит по DM или треду и хочет вернуться к предыдущему каналу.
- **Полное имя и роль пользователя** — в попапе инфо о канале (T8) показывать `first_name last_name` и роль (`@username — Full Name, Admin`) рядом с именем. Требует добавить поля `FirstName`, `LastName`, `Roles` в тип `User` и запрашивать их через уже используемый `GET /users/ids`.
- **Поиск сообщений (M4)** — детальная проработка: FTS через Mattermost REST `/search`, UI панели результатов.
- **Поддержка нескольких серверов** — профили в конфиге, переключение между серверами.
- **Уведомления ОС** — системные нотификации (`osascript` / `notify-send`) при упоминании пользователя.
- **Фильтр в ленте (M1)** — возможность фильтровать общую ленту по каналу без переключения в M2-режим.
- **Группы каналов в боковой панели** — сворачиваемые группы (Channels, Direct Messages, Favorites) аналогично десктопному клиенту Mattermost. Включает кастомные группы, задаваемые пользователем.
- **Закреплённые (pinned) каналы** — выделенная секция «Pinned» в верхней части списка каналов; поддержка закрепления/открепления через команду или горячую клавишу.
