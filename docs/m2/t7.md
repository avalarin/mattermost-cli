## Key decisions

**Archived channels always visible by default**: The user clarified that archived channels must always appear in the sidebar (not hidden behind a config option). The `ArchivedOnly` filter is an opt-in mode to show *only* archived channels, analogous to `UnreadOnly`.

**`rawItems` as canonical source for `WithSortAndFilter`**: The previous implementation iterated over `cv.items` which accumulated the effects of previous filter calls. After the first `WithSortAndFilter`, items already hidden by `UnreadOnly` filter would not be recoverable when the filter was toggled off. Using `rawItems` (the full, unfiltered set set once at construction) makes every re-filter idempotent.

**`ArchivedOnly` in `ChannelsConfig` (not `DebugConfig`)**: Archived-channel visibility is a user preference, not a developer toggle. Placing it in `[channels]` alongside `sort` and `unread_only` is the correct location.

**Filter popup: Unread and Archived on same row**: Both are boolean toggles with ←/→ navigation, so placing them side-by-side keeps the popup compact without adding an extra row.

**`ApplyDMNames` updates both lists**: DM display names are resolved asynchronously after `NewChannelsView`. If only `items` is updated, the next `WithSortAndFilter` call (reading from `rawItems`) would show the old DM channel names.

## Deviations from spec

- Original spec had `ShowArchivedChannels` in `DebugConfig` as an opt-in feature. After user feedback, the design changed: archived channels are always visible; `ArchivedOnly` is an opt-in *filter* analogous to `UnreadOnly`.

## Known limitations

- Archived channels loaded from the Mattermost server API already have `DeleteAt > 0`; there is no explicit "fetch archived channels" call. If the server omits archived channels from the team channel list, they simply won't appear even with the archived-only filter active.
