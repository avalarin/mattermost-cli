# M2-T5: Счётчики непрочитанных + пометка прочитанным

## What was done

- Added `ChannelUnread` type to `internal/mattermost/types.go`
- Added `GetChannelUnreads(channelID)` (GET `/users/me/channels/{id}/unread`) and `MarkChannelRead(channelID)` (POST `/channels/{id}/members/me/view`) to `internal/mattermost/client.go`
- Added `MsgUnreadsLoaded` and `MsgChannelRead` to `internal/tui/msgs.go`
- Added `unreadCounts map[string]int` field to `Model`
- Added `loadUnreadsCmd` (parallel fetch via goroutines) and `markReadCmd` (fire-and-forget POST) to `model.go`
- `Init()` now calls `loadUnreadsCmd` alongside `resolveDMNames`
- `handlePostedEvent` increments `unreadCounts[channelID]` when a message arrives in a non-active channel (only when a specific channel is open, not All Activity)
- `MsgChannelSelected` captures `prevChannelID` before overwriting `activeChannelID` and fires `markReadCmd(prevChannelID)` when switching from a non-empty channel
- `MsgUnreadsLoaded` handler populates `m.unreadCounts`
- `MsgChannelRead` handler resets `m.unreadCounts[channelID] = 0`
- `ChannelsView` gained `unreadCounts map[string]int` field, `SetUnreadCounts()` setter, and `renderLabel()` method that appends ` (N)` badge when count > 0; All Activity never gets a badge
- `renderBody()` in `model.go` now calls `SetUnreadCounts(m.unreadCounts)` in the channels view render chain

## Key decisions

- **Parallel fetch at startup**: `loadUnreadsCmd` uses goroutines + `sync.WaitGroup` to fetch unreads for all channels concurrently, minimising startup latency.
- **MarkChannelRead on channel switch away** (not on scroll-to-bottom): The spec decisions block (line 30) specifies "при уходе из канала", so mark-read fires when `MsgChannelSelected` processes a new channel, not when the viewport reaches the bottom. This is simpler and matches spec intent.
- **No increment in All Activity**: When `activeChannelID == ""`, all channels are being viewed, so no unread counter is incremented. The spec only increments for "не-активный канал".
- **Badge rendering in `renderLabel`**: The count suffix is computed first so the channel name can be truncated to fit within `cv.width` with the badge appended outside the truncation window.
- **Render-time injection via `SetUnreadCounts`**: Consistent with the existing pattern of passing render-time state (active, fg/bg colors) via value-receiver setters in the render chain. The counts are not stored persistently on the view between frames.

## Deviations from spec

None significant. The spec's US-5 AC mentions "прокручивает ленту до последнего сообщения" for mark-read, but the Decisions block (line 30) overrides this to "при уходе из канала". Implemented per decisions block.

## Known limitations

- `loadUnreadsCmd` fires one goroutine per channel. For workspaces with hundreds of channels this could be a burst of concurrent HTTP requests. A semaphore-limited pool would be better for large workspaces, but is out of scope for M2.
- There is no re-fetch of unreads after reconnect; the counter accumulates only from WS events after startup.
