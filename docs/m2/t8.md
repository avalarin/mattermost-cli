## Key decisions

- Popup follows the `*Popup` nil-means-closed pattern used by `threadPopup` and `searchPopup`.
- `i` only works when a real channel is selected (not All Activity); pressing `i` on All Activity is silently ignored.
- Members are loaded asynchronously via `loadMembersCmd`; popup shows "Loading..." until `MsgChannelMembersLoaded` arrives.
- Stale `MsgChannelMembersLoaded` responses (popup closed before they arrive) are silently dropped — the nil guard in the handler is sufficient.
- Both Esc and Enter close the popup and return to ModeChannels (symmetrical with thread popup).
- Popup dimensions: width capped at 64, height = feedH−4 (min 8). Centered via `lipgloss.Place`.

## Deviations from spec

None.

## Known limitations

- Members list is flat, one line, separated by two spaces. Long member lists will be truncated by lipgloss width constraints rather than wrapped into multiple lines.
- `per_page=100` means channels with >100 members show only the first 100.
