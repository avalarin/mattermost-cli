# M2 T6: Unified Search Popup (Ctrl+K)

## What was done

Implemented `SearchPopup` — a unified Ctrl+K overlay that combines channel/user search with sort/filter controls. Replaced the previous `ChannelFilterPopup` (which was a standalone sort/filter popup) with this new unified component.

**New files / major changes:**
- `internal/tui/channel_filter_popup.go` — completely rewritten; now contains `SearchPopup` (the old `ChannelFilterPopup` type is gone)
- `internal/tui/channel_filter_popup_test.go` — rewritten with new `SearchPopup` unit tests and model integration tests
- `internal/tui/channels_view.go` — `ChannelFilterState`/`ChannelSortOrder` types moved here (they belong with channel sorting logic); added `ChannelList()` method for extracting resolved channel list
- `internal/tui/model.go` — `ModeChannelFilter` → `ModeSearch`, `channelFilterPopup` → `searchPopup`, added `openSearchPopup`/`closeSearchPopup`/`handleKeySearch`, `searchCmd`, `findOrCreateDMCmd`; handles `MsgSearchResults`
- `internal/tui/msgs.go` — added `MsgSearchResults`
- `internal/tui/keys.go` — renamed `ChannelFilter` → `Search` (same Ctrl+K binding)
- `internal/mattermost/client.go` — added `SearchChannels` (GET) and `SearchUsers` (POST)
- `internal/mattermost/client_test.go` — added `TestSearchChannels_OK` and `TestSearchUsers_OK`

## Key decisions

**Unified popup instead of two separate popups.** The spec originally described a separate search popup, but the existing code already had a sort/filter popup on Ctrl+K. The user confirmed they wanted a single unified overlay. This is better UX: one place for channel navigation.

**Query < 2 chars → local mode.** Shows all local channels (from the channels sidebar, with resolved DM names), sorted/filtered by the current filter state. Filter controls are visible in the bottom section. Tab switches focus between results and filter rows, Space toggles the focused filter option.

**Query >= 2 chars → REST search mode.** Fires `SearchChannels` + `SearchUsers` concurrently in a single `tea.Cmd`. Filter controls hidden. Results: All Activity + channels + users.

**`ChannelList()` on `ChannelsView`.** Needed so the search popup can use channels with already-resolved DM display names. Extracting from the view avoids duplicating DM name resolution logic.

**Ctrl+C in ModeSearch closes popup** (same pattern as ModeHelp). The ModeSearch Ctrl+C interception is placed before the global Ctrl+C → exit handler.

**Filter applies on close with selection; Esc restores original.** Consistent with the old ChannelFilterPopup behavior. Any filter changes are rolled back on Esc.

## Deviations from spec

None. The spec described exactly this behavior. The unified popup was proposed by the user and approved before implementation.

## Known limitations

- No debounce on search input: each keystroke that crosses the 2-char threshold fires a REST request. Acceptable for now; the UI is non-blocking (results arrive asynchronously).
- Unread-only filter in local mode relies on `unreadCounts` passed at popup-open time and is not refreshed while the popup is open.
- `d` (delete) action in ModeThread remains a stub from T4 ("Not implemented yet"); unchanged by this task.
