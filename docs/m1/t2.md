# T2: GitHub Actions CI

## What was done

- `.github/workflows/ci.yml` — triggers on push and PR to `main`
- Steps: `actions/checkout@v4`, `actions/setup-go@v5` (Go 1.23 with built-in module cache), build, vet, test, lint
- Lint via `golangci/golangci-lint-action@v6` (latest version)

## Key decisions

**`actions/setup-go@v5` built-in cache**: `setup-go@v5` has native module and build cache support via `cache: true`, so no separate `actions/cache` step is needed. The spec mentioned `actions/cache` explicitly, but the built-in cache is equivalent and simpler.

**Separate vet step**: Added `go vet ./...` as an explicit step between build and test, mirroring the local `just check` workflow. Not in the spec but consistent with `CLAUDE.md`.

**`golangci-lint-action@v6`**: Handles installation and caching of golangci-lint internally. Using `version: latest` to stay in sync with the locally installed version.

## Deviations from spec

- CI uses `go vet` directly rather than via `make vet` / `just vet` — no need to install `just` in CI.
- Module cache handled by `setup-go`'s built-in `cache: true` rather than a manual `actions/cache` step.

## Known limitations

- CI can only be verified after pushing to GitHub — local simulation via `just check` covers the same steps.
