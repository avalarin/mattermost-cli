# T1: Project scaffolding + app skeleton

## What was done

- `go.mod` with module `github.com/avalarin/mattermost-cli`, Go 1.23, all required dependencies
- `go.sum` generated by `go mod tidy`
- `Makefile` with `build`, `test`, `lint` targets
- `.golangci.yml` with `errcheck`, `govet`, `staticcheck` (golangci-lint v2 format)
- Package stubs: `internal/config`, `internal/mattermost`, `internal/store`, `internal/tui`, `internal/tui/views`
- Full Bubble Tea TUI model in `internal/tui/model.go` with `ModeNormal`/`ModeCommand`, key handling, viewport, input
- Entry point `cmd/mattermost-cli/main.go` with `--config` and `--debug` flags
- Tests: `cmd/mattermost-cli/main_test.go`, `internal/tui/model_test.go`

## Key decisions

**golangci-lint v2 config format**: The installed golangci-lint is v2.8.0, which requires `version: "2"` in the config and uses `linters.default: none` instead of `disable-all: false`. The spec suggested the v1 format, which was updated to v2.

**Dependency versions**: `go mod tidy` resolved versions to latest at time of tidy — bubbletea v1.3.10, bubbles v1.0.0, lipgloss v1.1.0. The spec requested v1.x which is satisfied.

**Viewport initialization**: The viewport is lazily initialized on the first `tea.WindowSizeMsg` to avoid a zero-size viewport panic. Before that, `View()` returns "Initializing...".

**`StatusMsg()` getter**: Exposed as a public method on `Model` so tests can inspect status bar content without making `statusMsg` a public field.

**`internal/store/db.go`**: Stub returns `fmt.Errorf("database path must not be empty")` for empty path rather than `fmt.Errorf("not implemented")`, to pass errcheck without importing unused packages. The field `path` is stored for future use.

**`internal/tui/views/feed.go`**: FeedView is a standalone struct for future extraction but is not used by `model.go` in T1 — model embeds the viewport directly to keep T1 minimal. FeedView will be wired in T5 or later.

## Deviations from spec

- `.golangci.yml` uses golangci-lint v2 format (required by the installed toolchain); the spec showed v1 format.
- `gorilla/websocket` and `modernc.org/sqlite` appear in `go.mod` but are not imported yet (stubs don't use them), so they are not in `go.sum` after `go mod tidy`. They will be added when actually imported in T4/T7.
- `internal/tui/views/feed.go` is compiled but not wired to the model — it exists as a stub per the spec requirement.

## Known limitations

- `config.Load()` is a stub returning an empty Config — real TOML loading is T3.
- All `mattermost.Client` methods return `fmt.Errorf("not implemented")` — REST implementation is T4.
- `store.DB` and `store.Store` are field-only stubs — SQLite schema is T7.
- WebSocket client is a struct stub — reconnect logic is T6.
- The feed viewport shows a static placeholder — live messages are T6.
